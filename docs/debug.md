# 为什么电池电量耗尽，插入USB 会一直进入下载模式？
~~~
原因分析：这是电池低压 + USB 插入时的电源切换 + 自动下载电路共同导致的典型问题。核心原因自动下载电路（Auto Boot/Reset Circuit）
大多数 ESP32-S3 开发板（尤其是带 USB-UART 芯片如 CP2102、CH9102 的）都有 DTR/RTS 自动控制 GPIO0（BOOT）和 EN（RESET）的电路。
当你插入 USB 时，USB-UART 芯片上电，会产生一个复位脉冲 + GPIO0 拉低的动作，让芯片自动进入 Download Mode（boot:0x0）。
这本来是方便烧录设计的，但低电压下表现异常。

电池电压只有 3.45V（偏低）  LiPo 电池标称 3.7V，3.45V 已接近欠压保护边缘（很多保护板在 3.2~3.4V 就会切断）。  
ESP32-S3 正常工作电压 3.0~3.6V，3.45V 勉强够用，但负载能力变差，电压容易跌落。  
插入 USB（5V）瞬间，板子上电源切换电路（二极管或 PMOS）会从电池切换到 USB 供电。这个切换过程会产生瞬间电压波动/毛刺，导致 EN 或 GPIO0 信号不干净，芯片复位后直接卡在 Download Mode。

为什么按复位键才能退出？
进入 Download Mode 后，芯片就停在等待烧录状态。只有硬件复位（EN 拉低） 并且 GPIO0 不是低电平 时，才会正常启动用户程序。
低压 + 电源切换导致的复位信号不彻底，所以一直卡住，直到你手动按复位键（强行干净复位）。

验证与解决办法立即解决（临时）：插入 USB 前先按住 BOOT 键（GPIO0），再插 USB，然后松开 → 看是否稳定。
或者插 USB 后直接按一下复位键，观察是否正常启动。
电池充满电（>3.8V）再测试，问题大概率消失。

长期解决：给电池充电 或换一块容量/电压更高的电池。
在电源切换电路加电容（很多玩家在 3.3V 或 EN 脚加 10~47uF 电容，改善切换时的毛刺）。
修改固件：在 setup() 最前面加延迟或检测 USB 插入（用 USB.vbusPresent()），避免低压下异常行为。
硬件优化（推荐）：用带更好电源管理的板子（如某些 XIAO ESP32-S3 或 Adafruit Feather S3，有更完善的 USB/Battery 切换）。
或者在电池和板子间加低压保护/升压模块（确保始终稳定 3.3V）。

如果你经常这样操作，烧录前手动进入下载模式（按住 BOOT + 按复位），烧完后再按复位退出。

把你的具体开发板型号（比如 LilyGo、XIAO、自定义板等）告诉我，我可以给你更针对性的电路或代码建议。
另外，电池长期在 3.45V 以下运行不推荐，容易损伤电池和芯片稳定性。充满电再用会好很多！

也就是电池在低压没电的时候，插入USB可能会存在触发芯片进入下载模式。现象就是你怎么插USB跟按按键都没用。只有充电到3.5V以上然后按复位键的时候才会重新开机。
