
# 自动打包工具

- **pos_extra_script.py** 将编译生成的 boot.bin、bootload.bin、partition.bin 打包生成一个 bin 文件；

应用举例：在 platformIO 的环境中，找到 platformio.ini 配置文件，在配置文件关于工程的默认环境下，添加该脚本，如下：

~~~
...

[env]
platform = espressif32@6.3.0
framework = arduino
upload_speed = 115200
...
extra_scripts =  ./script/pos_extra_script.py
...
~~~

如上所示，在添加 extra_scripts 后，在编译整个工程，编译完成后会自动调用 pos_extra_script.py 脚本生成 firmware_factory_xxxxxxxx.bin 固件；

如果该工程下还有 data 文件夹，则会自动的把 data 文件识别为 spiffs 文件系统，并将 data 文件夹下的所有文件打包成 spiffs.bin，还会自动生成 partitions.csv 文件；

**pos_extra_script.py 生成的所有文件都会放在 "工程目录/build/" 下面；**


# SPIFFS 文件系统

SPIFFS 是一个用于 SPI NOR flash 设备的嵌入式文件系统，支持磨损均衡、文件系统一致性检查等功能。

- **spiffsgen.py** 文件系统打包脚本

~~~ python
python spiffsgen.py <image_size> <base_dir> <output_file>

参数（必选）说明如下：
image_size：分区大小，用于烧录生成的 SPIFFS 镜像；
base_dir：创建 SPIFFS 镜像的目录；
output_file：SPIFFS 镜像输出文件。
~~~

应用举例：将工程 T-Embed-PN532 文件下的 data 文件打包，生成的 .bin 文件放到 build 文件夹下，则因向命令行输入如下命令：

~~~ python
python spiffsgen.py 0x360000 ..\data\ ..\build\spiffs.bin
~~~

## build_example_firmware

作用是从 `examples/` 里选示例编译，并把可下载的固件文件导出到 `firmware/examples/<示例名>/bin/`。

最常用的几种命令是：

列出所有可构建示例。
~~~powershell
python script\build_example_firmware.py --list
~~~

构建单个示例。
~~~powershell
python script\build_example_firmware.py test_ir_send_recv
~~~

一次构建多个示例。
~~~powershell
python script\build_example_firmware.py test_ir_send_recv test_ws2812
~~~

一次构建 examples/ 下全部示例。
~~~powershell
python script\build_example_firmware.py --all
~~~

可选参数：
~~~powershell
python script\build_example_firmware.py test_ir_send_recv --no-clean
python script\build_example_firmware.py test_ir_send_recv --no-merge
python script\build_example_firmware.py --all --continue-on-error
~~~

参数说明：
- `--no-clean`：跳过构建前清理，加快二次编译
- `--no-merge`：不生成合并后的单文件 bin
- `--continue-on-error`：某个示例失败后继续构建后面的示例
- `--env T_Embed_CC1101`：指定 PlatformIO 环境
- `--chip esp32s3`：指定烧录命令里的芯片型号

构建完成后，输出目录类似：
~~~powershell
firmware\examples\test_ir_send_recv\bin\
~~~

里面通常会有：
- `bootloader.bin`
- `partitions.bin`
- `firmware.bin`
- `test_ir_send_recv.bin` 这是合并后的单文件固件
- `esptool_command.txt`
- `esptool_merged_command.txt`
- `build_info.json`

如果你想直接烧录合并包，可以看对应目录下的 `esptool_merged_command.txt`。
补一句，最好不要同时开两个 `build_example_firmware.py` 进程，因为它们会共用 `.pio/build/T_Embed_CC1101`。
